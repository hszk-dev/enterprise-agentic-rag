# Architecture Decision Records (ADR)

このディレクトリには、プロジェクトの重要なアーキテクチャ決定を記録します。

## ADRとは？

ADR (Architecture Decision Record) は、ソフトウェアアーキテクチャに関する重要な決定を記録するためのドキュメントです。
決定の背景、検討した選択肢、最終的な決定とその理由を明確に記録します。

## 使い方

### 新しいADRの作成

1. `000-template.md` をコピー
2. ファイル名を `YYYY-MM-DD-title.md` 形式にリネーム
3. テンプレートに従って内容を記入

### ステータス

- **Proposed**: 提案中（レビュー待ち）
- **Accepted**: 承認済み（実装可能）
- **Deprecated**: 非推奨（より良い方法が見つかった）
- **Superseded**: 置き換え済み（新しいADRを参照）
